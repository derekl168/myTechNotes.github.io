Single Command FreeIPA Lab Provisioning through Vagrant+Ansible Automation

Posted on 2020-01-21 In blog

FreeIPA is a free and open source identity management system, it is also the upstream open-source project for Red Hat Identity Manager (IdM). FreeIPA combines Linux (Fedora), 389 Directory Server, MIT Kerberos, NTP, DNS, Dogtag Certificate System), and SSSD, to provide a centrally managed Identity, Policy, and Audit (IPA) system.

We use Red Hat Identity Manager (IdM) in one of our projects as a centralised authentication and Single-Sign-On (SSO) system to manage Linux Servers. Recently, in order to troubleshoot a technical issue, I try to build a local lab on my laptop so I can rapidly spawn and teardown the test lab for testing different technical approaches. I decide to use the upstream FreeIPA project to re-produce the issue, as it’s just a personal lab.

The following is the quick procedure I use to setup my lab on laptop. I use FreeIPA’s ansible roles on top of vagrant ansible provider. Through this way, the whole FreeIPA environment setup, including ipa server, client and the Integration, is literally just “one-command”.

Preparation

Network Planning

Before start, I draw a quick plan for the testing lab:

IPA Server

  • FQDN: idm-server.my.domain
  • IP: 192.168.33.20

IPA Client

  • FQDN: idm-node1.my.domain
  • IP: 192.168.33.21

I will still use default vagrant network to access Internet, and provision the 2nd private network for the communication between IPA server and client.

Setup Overview

The idea is to fully automate the IPA server & client VM / OS provisioning, as well as all the software installation and configuration.

So, basically, I will use vagrant to provision two Centos7 VM with the static IP addresses which are planned as the above-mentioned. As vagrant dynamically assigns the port for VM port-forwarding, I will also leverage vagrant’s ansible provider to automatically generate the required ansible inventory file.

The IPA installation will be done by ansible freeipa roles - ipaserver and ipaclient. Ansible playbook will be run by Vagrant ansible provider. The overall testing planning is depicted as the following tree:

Ansible playbook

Through the FreeIPA, the required playbooks are actually very simple, as follows:

derek@dhcp-65-103 provisioning % cat preparation-server.yml

Notes:

FreeIPA requires FQDN so I remove hostname’s loopback address 127.0.0.1

By default, FreeIPA expects you to setup firewall which is not included in Centos7 cloud image, unless you purposely disable firewall in freeipa configuration parameter.

Another small tip is to include the yum update before IPA installation to address the bug in old nss package.

The IPA client playbook is also similar.

IPA Deployment

Once the preparation is down, the IPA server and client VM provisioning and software installation / configuration are just a single command:

derek@dhcp-65-103 idm-lab % vagrant up
